Explain The Attack
מתקפת Reflected XSS
הזרקנו את הקוד הזדוני כקלט לפרמטר msg ב- URL של הדף הפגיע task1ref.php, והדף שיקף את 
הקלט חזרה ל- HTML ללא סינון או בדיקה, ולכן הקוד הורץ באופן מיידי בעת שליחת ה־URL וטעינת 
הדף בדפדפן הקורבן. הרצת הקוד גרמה לדפדפן של הקורבן לבצע בקשת HTTP לשרת התוקף, שלכד 
את ה- cookie של המשתמש. לאחר מכן נעשה בו שימוש כדי לשלוח בקשת HTTP מזויפת לדף המוגן 
gradesPortal.php, ובאמצעותה התקבלה הגישה לתוכן המוגן ול- flag.


מתקפת Stored XSS 
הקוד הזדוני נשלח לשרת כערך של השדה comment באמצעות בקשת POST לדף 
task2stored.php, ונשמר במסד הנתונים. בניגוד ל־Reflected XSS, הקוד אינו מורץ מיידית, אלא רק 
כאשר משתמש נכנס לדף AdminViewComments.php אשר מציג את התגובות מהמסד Mariadb.
בעת הצגת הנתונים הקוד מורץ בדפדפן של הקורבן וגורם לשליחת בקשת HTTP לשרת התוקף 
שמכילה את ה- cookie של המשתמש. ה- cookie שנלכד משמש את שרת התוקף לגישה לדף המוגן 
GradersPortalTask2.php ולקבלת ה־flag.


מתקפת DOM - based XSS
במתקפה הזאת לא הייתה שום מעורבות של השרת בעיבוד הקלט. הקוד הזדוני הוכנס כקלט ב- URL
 fragment (#) (מיד לאחר הפרמטר user) של הדף הפגיע task3dom.php. הקלט נקרא ע״י ה-
JavaScript בצד הלקוח, ומוזרק ישירות ל- DOM בצורה לא בטוחה, מה שגורם להרצת קוד בדפדפן 
מיד עם טעינת הדף, ששולח בקשת HTTP לשרת התוקף שמכיל את ה- cookie של המשתמש. 
לאחר מכן שרת התוקף שולח בקשת HTTP עם אותו ה- cookie לדף המוגן 
studentManagerDOMBASED.php ומקבל גישה ל- flag.
Attacker's Server
שרת התוקף נבנה כשרת TCP שמאזין ל- port מוגדר, מקבל בקשות HTTP נכנסות מהדפדפן שנוצרו 
מהקוד הזדוני, ומחלץ מהבקשות את ה- cookie של המשתמש. השרת השתמש ב- session cookie 
של המשתמש כדי לבנות בקשות HTTP שלכאורה מגיעות מהמשתמש כדי לגשת למידע מוגן ול- flag מתוך האתר.
